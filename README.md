Код тестирует API адреса и использует при каждой проверке нового, созданного пользователя, который потом удаляется через профиль admin дабы не засорять базу данных.

Автотесты API сервиса secby.ru
## Проверки

- Авторизация пользователя
- Получение токена
- Получение профиля пользователя
- Проверка ограничения доступа user -> admin
- Проверка доступа admin -> admin

fixture `test_user`:

1. Создаёт тестового пользователя через API регистрации
2. Получает `id` созданного пользователя
3. Передаёт данные пользователя в тесты
4. После выполнения теста удаляет пользователя через профиль admin

Используется конструкция `yield`, чтобы удаление профиля выполнялось после завершения теста.

`test_login_api`:

Проверяет:

1. Успешный login пользователя
2. Получение токена
3. Ответ API

`test_get_user_profile`:

1. Проверяет получение собственного профиля через API "GET /api/profiles/{account_id}"

`test_user_cannot_access_admin_profile`:

1. Негативный тест. Проверяет, что обычный пользователь не может получить профиль администратора.

`test_admin_can_access_admin_profile`:

1. Проверяет, что администратор может получить информацию о собственном профиле.

`test_change_password`:

1. Проверяет возможность смены пароля

`test_get_list`:

1. Проверяет получение списка пользователей с помощью аккаунта администратора

`test_nonexistent_profile`:

1. Негативный тест. Проверяет Get Profile с несуществующим `id`

`test_wrong_password`:

1. Негативный тест. Проверяет ошибку при Login с неверными данными
